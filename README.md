# BinGeo-LDM++

BinGeo-LDM++ 是一个基于扩散模型的双目深度估计算法框架，特别针对 KS861 单 USB 复合双目相机优化，在保持实时性的同时提供高精度深度估计。

## 项目概述

本项目基于 RollingDepth 模型进行改进，实现了四个核心创新点，专门针对 KS861 相机的硬件特性和 15fps 的实时性要求进行优化，能够在中端 GPU 上实现高效稳定的深度估计。

## 核心创新点

### 1. KS861 内参约束的双目交叉帧扩散模型

- 将 KS861 相机的精准内参（焦距、主点、畸变系数）嵌入扩散模型条件层
- 通过内参引导的极线注意力机制约束双目视差一致性
- 解决单目扩散模型几何歧义问题，同时保留实时推理能力

### 2. KS861 实时性适配的轻量化分层片段采样

- 基于运动强度的动态分层采样策略
- 静态帧（运动强度低）采用大间隔采样（降低计算量）
- 动态帧（运动强度高）采用小间隔采样（保留细节）
- 采样间隔与 KS861 的帧率锁定，确保 15fps 实时输出

### 3. 深度置信度引导的动态权重全局对齐

- 结合 KS861 的双目一致性与深度不确定性，为每个片段分配动态权重
- 高运动强度片段降权、高置信度片段升权
- 加入时间平滑约束，避免 15fps 下的深度跳变

### 4. KS861 噪声适配的深度不确定性估计

- 通过扩散模型多步采样方差估计深度不确定性
- 低置信度区域（如噪声污染区域）动态补全
- 提升 15fps 下的深度鲁棒性

## 安装指南

### 1. 环境准备

```bash
# 克隆项目（假设已在本地）
cd d:/AnacondaProjects/BinGeo-LDM++

# 安装依赖
pip install -r requirements.txt
```

### 2. 配置相机参数

确保相机内参文件已正确放置在 `configs/` 目录下：
- `ks861.txt`：KS861 相机内参（txt 格式）
- `ks861.yaml`：KS861 相机内参（yaml 格式）

这些文件包含相机的焦距、主点、畸变系数、基线长度等关键参数，用于模型的内参约束和极线计算。

## 使用说明

### 实时深度估计

运行 KS861 相机专用的实时深度估计程序：

```bash
python run_ks861_realtime.py
```

程序会自动加载相机配置、初始化模型，并启动实时深度估计流程。程序采用流水线并行处理架构，确保 15fps 的实时性能。

### 主要功能

- 实时获取 KS861 相机的双目图像
- 动态分层采样（基于运动强度）
- 内参约束的双目交叉帧扩散模型推理
- 深度置信度引导的动态权重全局对齐
- 噪声适配的深度不确定性估计与低置信度区域补全
- 实时显示 RGB 图像与深度图
- 支持保存深度估计结果

## 技术细节

### 模型架构

- 基于 RollingDepth 的 Marigold 基础架构，扩展为双目输入
- 条件编码器中加入内参嵌入层
- Transformer 块中使用极线约束注意力机制
- UNet backbone 进行通道剪枝优化（保留 60% 通道）
- 支持 FP16 混合精度推理

### 实时优化策略

- 流水线并行：图像采集→预处理→片段采样→扩散推理→对齐补全
- 多线程并行处理，每个阶段耗时 ≤13ms
- 总推理时间 ≤65ms/帧，满足 15fps 实时要求
- 内存占用优化，峰值内存 ≤8GB

### 训练配置（预留）

- 硬件环境：单 GPU（RTX 4090，24GB），CPU Intel i9-13900K，内存 64GB
- 训练参数：batch size=8，优化器 AdamW（初始学习率 3e-5）
- 训练轮次：80 轮（前 50 轮训练基础模型，后 30 轮加入不确定性损失）
- 损失函数：扩散噪声预测损失 + 双目视差一致性损失 + 不确定性波动损失

## 注意事项

- 确保 KS861 相机已正确连接并安装驱动
- 运行前检查 configs 目录下的相机内参文件是否完整
- 为达到最佳性能，建议在支持 CUDA 的 GPU 上运行
- 默认配置已针对 15fps 实时性进行优化
